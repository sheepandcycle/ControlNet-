# 项目运行
## 无法通过默认方式联网下载CIP模型
### 问题及原因  
出现了"Can't load tokenizer for 'openai/clip-vit-large-patch14'"的报错，原因是“既无法联网下载，本地也没有对应模型文件”。这是由于代码中默认是联网下载CLIP模型，但由于网络原因下载时遇到了错误，导致tokenizer加载失败。  
### 解决方法  
手动下载CLIP模型的相关文件（vocab.json、merges.txt、tokenizer_config.json、special_tokens_map.json）并放到相应路径（ControlNet-main/models/clip-vit-large-patch14）。  
将"ldm/modules/encoders/modules.py"文件中
~~~ python
self.tokenizer = CLIPTokenizer.from_pretrained(version)
self.transformer = CLIPTextModel.from_pretrained(version)
~~~
改成
~~~ python
self.tokenizer = CLIPTokenizer.from_pretrained("./models/clip-vit-large-patch14")
self.transformer = CLIPTextModel.from_pretrained("./models/clip-vit-large-patch14")
~~~
让tokenizer和CLIPTextModel都由默认联网下载改为本地
## 项目运行后无法生成图像
### 问题及原因
后端模型已经加载完成、Gradio服务启动成功后在UI界面上传图像及prompt，但一直显示processing，无法生成图像。  
先写了一个完全不依赖模型的脚本（test_controlnet_gradio.py）测试前后端通信是否正常
~~~python
import gradio as gr
from PIL import Image
import numpy as np

# 这是一个模拟推理函数，不用模型，直接返回灰度图
def dummy_inference(image, prompt):
    print(f"Received prompt: {prompt}")
    print(f"Received image size: {image.size}")
    gray = image.convert("L")  # 转灰度
    return gray

# Gradio 界面
demo = gr.Interface(
    fn=dummy_inference,
    inputs=[
        gr.Image(type="pil"),
        gr.Textbox(label="Prompt")
    ],
    outputs=gr.Image(type="pil"),
    title="ControlNet Test"
)

# 启动
demo.launch(server_name="127.0.0.1", server_port=7860, share=False)
~~~
结果UI界面显示生成图片的地方出现"error"字样，怀疑是gratio版本（gradio 3.16.2）问题，升级到3.41.2后，脚本可以正常运行生成灰度图。  
但运行controlnet本身时却又在UI界面显示生成图片的地方出现"error"字样。怀疑是队列通信异常，于是将gradio_canny2image.py代码中
~~~python
block = gr.Blocks().queue()
~~~
改为
~~~python
block = gr.Blocks()
~~~
成功解决问题
